# Реліз v0.8.0

`v0.8.0` вирішує практичну проблему: після кожного перезавантаження сторінки доводилося знову завантажувати файли і знову вводити параметри. Тепер проби, холості, сурогати і налаштування зберігаються в браузері — відкрив і продовжив роботу з того місця, де зупинився.

## Що змінилося для користувача

- **LabSpaces**: новий workspace для керування всіма даними в одному місці:
  - Зберігайте blank, surrogate та analysis datasets без повторного завантаження
  - Керуйте analyzer presets з параметрами скринінгу (RT tolerance, m/z tolerance, Signal-to-Blank thresholds)
  - Переглядайте та повторно відкривайте раніше створені звіти
  - Перемикайтеся між dataset-ами без втрати поточного стану
- **Persistent storage**: всі LabSpaces дані зберігаються локально в браузері:
  - Автоматичне збереження при кожній зміні
  - Миттєве відновлення стану після перезавантаження сторінки
  - Export / import LabSpaces для резервного копіювання або переносу між пристроями
- **Покращений workflow**:
  - Швидке перемикання між різними dataset-ами без повторного завантаження файлів
  - Повторне використання analyzer presets для порівняння результатів при різних параметрах
  - Доступ до історії звітів без повторного запуску обчислень

## Що це означає на практиці

- Завантажили dataset один раз — використовуєте скільки разів потрібно.
- Краща організація роботи з різними типами даних: blank, surrogate та analysis чітко розмежовані.
- Можна швидко порівнювати результати з різними параметрами скринінгу, не перезавантажуючи файли.
- Менше часу на рутинну повторну обробку.
